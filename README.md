# Java Linux

Java Linux is a high-performance meta-environment built for using multiple distro purposes. It uses the JRC (Java Runtime Controller) to manage system states with elite efficiency. Through its `jrc-hijack` method, it transforms any base into a unified, JVM-centric userland prioritizing x86-64-v4 speed and a highly customizable experience.

### ⚠️ CAUTION
Initiating `jrc-hijack` modifies your core system environment. 
- Always backup your `/etc/` and `/usr/` directories before use.
- Ensure your base is compatible

### Key Features
* **Distro Agnostic:** Seamlessly bridge Arch, Gentoo, and Debian binaries.
* **JRC Management:** High-efficiency system state control.
* **Optimized Core:** Native v4 instruction set utilization for modern CPUs.
* **Silk Aesthetic:** Fully customizable UI geared toward Google Sans and modern typography.

### 📊 Support Matrix

| Base Distribution | Support Level | Note |
| :--- | :--- | :--- |
| **Arch / Arch-Based** | 🟢 **BEST** | Native v4 support & seamless JRC integration. |
| **Gentoo / Gentoo-Based** | 🟢 **BEST** | Perfect for Silk-Runtime optimization. |
| **Void Linux** | 🟢 **BEST** | Runit/OpenRC compatibility is highly stable. |
| **Debian** | 🟡 **MILDLY GOOD** | Standard GLIBC paths; manual JRC linking required. |
| **Fedora** | ⚪ **MID** | SELinux often interferes with Hijack protocols. |
| **NixOS** | 🔴 **NOT SUPPORTED** | Filesystem hierarchy conflicts with Silk layer. |
