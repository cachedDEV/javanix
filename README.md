# Java Linux

Java Linux is a high-performance meta-environment built for using multiple distro purposes. It uses the JRC (Java Runtime Controller) to manage system states with elite efficiency. Through its `jrc-hijack` method, it transforms any base into a unified, JVM-centric userland prioritizing x86-64-v4 speed and a highly customizable experience.

### ⚠️ CAUTION
Initiating `jrc-hijack` modifies your core system environment. 
- Always backup your `/etc/` and `/usr/` directories before use.
- Ensure your base is compatible

### Key Features
* **Distro Agnostic:** Seamlessly bridge Arch, Gentoo, and Void binaries.
* **JRC Management:** High-efficiency system state control.
* **Optimized Core:** Native V instruction set utilization for modern CPUs.
* **Custom Aesthetic:** Fully customizable UI geared toward Google Sans and modern typography.

### 📊 Support Matrix

| Base Distribution | Support Level | Note |
| :--- | :--- | :--- |
| **Arch / Arch-Based** | 🟢 **BEST** | Native support & seamless JRC integration. |
| **Gentoo / Gentoo-Based** | 🟢 **BEST** | Perfect for Runtime optimization. |
| **Void Linux** | 🟢 **BEST** | Runit/OpenRC compatibility is highly stable. |
| **Debian** | 🟡 **MILDLY GOOD** | Standard GLIBC paths; manual JRC linking required, may have to use skgit |
| **Fedora** | ⚪ **MID** | Security often interferes with Hijack protocols. |
| **NixOS** | 🔴 **NOT SUPPORTED** | Nix is a advanced distro, We tried, and tried, but still no signs of it working. |
